# Configuration Reference

All pipeline behaviour is controlled via `.claude/settings.json`. This document describes every configuration option.

---

## Top-Level Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `model` | string | `claude-sonnet-4-6` | Claude model to use. Pin to a specific version. |
| `permissions.allow` | string[] | See settings.json | Bash commands Claude is permitted to run |
| `permissions.deny` | string[] | See settings.json | Bash commands that are hard-blocked |

---

## Environment Variables (`env`)

These are injected into every hook script as environment variables.

| Variable | Default | Description |
|----------|---------|-------------|
| `PIPELINE_STATE_FILE` | `.claude/state/pipeline.json` | Path to pipeline state file |
| `PIPELINE_LOG_FILE` | `.claude/state/pipeline.log` | Path to pipeline audit log |
| `COVERAGE_LINE_THRESHOLD` | `80` | Minimum line coverage % (0-100) |
| `COVERAGE_BRANCH_THRESHOLD` | `70` | Minimum branch coverage % (0-100) |
| `COMPLEXITY_FLAG_ABOVE` | `10` | Cyclomatic complexity above which to flag |
| `COMPLEXITY_BLOCK_ABOVE` | `20` | Cyclomatic complexity above which to block |
| `SAST_BLOCK_SEVERITY` | `HIGH` | Minimum SAST severity to block (HIGH or CRITICAL) |
| `MAX_AUTO_REMEDIATION_ATTEMPTS` | `2` | Remediation loop iterations before escalation |
| `MAX_CONTEXT_TOKENS` | `4000` | Max tokens of codebase context in generation prompts |
| `APPROVED_LANGUAGES` | `python,typescript,...` | Comma-separated list of approved languages |
| `HIGH_RISK_PATTERNS` | `authentication,...` | Comma-separated patterns triggering security review |
| `PERFORMANCE_REGRESSION_THRESHOLD_PCT` | `15` | % above NFR threshold that triggers a block |
| `PIPELINE_VERSION` | `1.0.0` | Pipeline schema version (do not change) |

## Hooks Configuration

Hooks run automatically at lifecycle events. The pipeline uses three hook types:

### `PreToolUse` — Before bash commands

Runs `.claude/hooks/pre-bash.sh` before any bash execution. Performs safety checks and logs sensitive operations.

### `PostToolUse` — After file writes

Runs `.claude/hooks/post-write.sh` after any Write/Edit/MultiEdit operation. Performs quick syntax checks and secret pattern detection.

### `Stop` — When Claude's session ends

Runs `.claude/hooks/on-stop.sh` to persist a checkpoint to the state file.

## Customising the Approved Stack

Edit `env.APPROVED_LANGUAGES` to add or remove languages:

```json
"env": {
  "APPROVED_LANGUAGES": "python,typescript,go,rust"
}
```

The pipeline's language-specific tool selection (linters, type checkers, test runners) is driven by the `target_language.language` field in the spec, cross-referenced against this list.

## Adjusting Quality Thresholds

For a higher-assurance environment:

```json
"env": {
  "COVERAGE_LINE_THRESHOLD": "90",
  "COVERAGE_BRANCH_THRESHOLD": "80",
  "COMPLEXITY_FLAG_ABOVE": "8",
  "COMPLEXITY_BLOCK_ABOVE": "15",
  "SAST_BLOCK_SEVERITY": "MEDIUM"
}
```

For a rapid-prototyping environment (not recommended for production):

```json
"env": {
  "COVERAGE_LINE_THRESHOLD": "60",
  "COVERAGE_BRANCH_THRESHOLD": "50",
  "MAX_AUTO_REMEDIATION_ATTEMPTS": "3"
}
```

## Adding Custom Semgrep Rules

Add rules to `.claude/skills/semgrep-rules.yaml`. The file uses standard Semgrep YAML rule syntax. Custom rules are run in addition to the auto-selected language rulesets, not instead of them.

See: https://semgrep.dev/docs/writing-rules/

---

## Permissions

The `permissions.allow` list uses glob patterns. The pipeline's default allows the tools required for analysis and generation. Do not remove entries without understanding their impact.

To allow a new tool globally:
```json
"permissions": {
  "allow": [
    "Bash(your-tool*)",
    ...
  ]
}
```

To restrict a currently-allowed tool, add it to `permissions.deny`. Deny rules take precedence over allow rules.

## State File Schema

`.claude/state/pipeline.json`:

```json
{
  "run_id": "uuid-v4",
  "started_at": "ISO-8601",
  "pipeline_version": "1.0.0",
  "last_checkpoint": "ISO-8601",
  "stages": {
    "stage1": {
      "status": "PASS | FAIL | IN_PROGRESS | ESCALATED | NOT_STARTED | APPROVED | REJECTED",
      "timestamp": "ISO-8601 | null",
      "score": "string | null",
      "findings": [
        { "timestamp": "ISO-8601", "message": "string" }
      ]
    }
  },
  "metrics": {}
}
```

Do not edit this file manually. Use `bash .claude/hooks/update-state.sh` or the pipeline commands.
