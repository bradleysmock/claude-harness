# Solution

**Ticket**: 0010
**Title**: Changelog generator

## Approach

Add a `commands/changelog.md` command file. When invoked, Claude runs a sequence of shell commands to find the last git tag, collect completed ticket titles and conventional commits since that tag, categorize and deduplicate them, and render a Markdown block that is prepended to (or replaces the unreleased block in) `CHANGELOG.md`. All logic uses bash with strict safety requirements (see Shell Safety below). No new Python modules or runtime dependencies are introduced.

## Components

| Component | Responsibility | Shell safety constraints |
|-----------|----------------|--------------------------|
| `commands/changelog.md` | Main command; all bash runs under `set -euo pipefail` | global |
| Boundary detection | `git describe --tags --abbrev=0`; fall back to `git rev-list --max-parents=0 HEAD`; abort (`exit 1`) if both fail; store in `"$boundary"` | double-quote `"$boundary"` in all downstream uses |
| Tag reference file | `tmpref=$(mktemp); trap 'rm -f "$tmpref"' EXIT; git log -1 --format='%at' -- "$tag" \| xargs -I{} touch -t "$(printf '%(%Y%m%d%H%M.%S)T' {})" "$tmpref"` (POSIX `touch -t`, not GNU `-d`) | `"$tag"` double-quoted; `"$tmpref"` via mktemp; failure is fatal (`\|\| { echo "error: cannot create tag ref file" >&2; exit 1; }`) |
| Ticket collector | Scan `.tickets/completed/*/status.md`; extract `title:` and `updated:` (YYYY-MM-DD); compare date-only against tag date (`git log -1 --format=%as "$tag"`); fallback to `find -newer "$tmpref"` when `updated:` absent or malformed — failure to create `$tmpref` is fatal before this step | `"$tmpref"` double-quoted; file paths quoted |
| Commit collector | `git log "$boundary"..HEAD --pretty=format:'%s'` | `"$boundary"` double-quoted |
| Sanitizer | For every external string (ticket title, commit subject) before any other processing: (1) strip `\r` and `\n`; (2) strip leading `#`; (3) escape Markdown structural chars: replace `[` → `\[`, `]` → `\]`, `(` → `\(`, `)` → `\)`, `<` → `\<`, `>` → `\>`. Result is safe to interpolate into a Markdown list item. | applied before categorizer, deduplicator, and formatter |
| Categorizer | Regex match `^(feat\|fix\|chore)(\(.*\))?:` on sanitized commit subject; slug-prefix match `^feat-\|^fix-\|^chore-` on ticket slug | pure string logic on sanitized inputs |
| Deduplicator | Normalize: (1) strip `^[a-z]+(\(.*\))?: ` prefix from sanitized commit subject; (2) lowercase; (3) trim leading/trailing whitespace only. If normalized ticket title equals normalized commit subject, keep ticket entry, drop commit entry | pure string; no mid-string punctuation stripping |
| Formatter | Render `## [Unreleased] - YYYY-MM-DD` (Keep a Changelog ASCII hyphen); populate `### feat/fix/chore/other` subsections; omit empty subsections; print runtime warning to stderr if replacing existing `[Unreleased]` block | string template on sanitized, deduplicated entries |
| CHANGELOG writer | Write to `$(mktemp)` first, then `mv` atomically. Boundary cases: (a) absent → create; (b) `^## \[Unreleased\]` present → replace from that line through EOF or through the blank line before the next `^## ` heading (if one exists); (c) no `[Unreleased]` block → prepend | atomic write; anchored regex at `^` |

## Shell Safety Requirements

All bash generated by the command must satisfy:
- `set -euo pipefail` at top of every script block.
- All variable expansions with user-controlled data (titles, subjects, paths, tag names) must be double-quoted.
- CHANGELOG write uses `mktemp` + `mv` — never truncate-in-place.
- Any `cd` is followed by `|| exit 1`.
- `touch -t` (POSIX) not `touch -d` (GNU) for portability on macOS.
- Block-replacement regex anchors `## [Unreleased]` at line start (`^`).
- `find -newer "$tmpref"` failure (non-existent ref file) must be fatal, not silently inclusive.
- Sanitizer is applied to all external strings **before** they are used in any comparison, categorization, or output step.

## Tech Choices

| Choice | Rationale |
|--------|-----------|
| Markdown command file (no new Python) | Consistent with all other harness commands; no new dependency surface |
| `git describe --tags --abbrev=0` for boundary | Canonical; handles annotated and lightweight tags; fail-closed if both git commands fail |
| Date-only comparison for ticket filtering | `updated:` is YYYY-MM-DD; tag date extracted as `%as` — same granularity, no timezone ambiguity |
| Slug-prefix heuristic for ticket type | No formal `category:` field in `status.md` yet; slug prefix is the only machine-readable signal |
| Prepend / replace, not append | Keep a Changelog: newest entries at top; replace-on-rerun provides idempotency |
| ASCII hyphen in heading separator | Keep a Changelog strict format; downstream parsers expect this |
| Sanitizer: escape Markdown structural chars | Defense-in-depth: `[Unreleased] - 2026-06-21` in a commit subject cannot mimic the block heading when `[` is escaped to `\[`; also neutralizes link syntax injection |
| `touch -t` (POSIX) over `touch -d` (GNU) | macOS BSD `touch` does not support `-d`; `-t [[CC]YY]MMDDhhmm[.SS]` is portable |
| `find -newer` reference file failure is fatal | Fail-closed: an undefined boundary must never silently include all tickets |

## Test Plan

| Requirement | Test Type   | Scenario(s) |
|-------------|-------------|-------------|
| FR-2 | Integration | Repo with one tag → boundary = tag SHA; repo with no tag → boundary = root commit; both git commands fail → command exits non-zero |
| FR-3 | Integration | 3 tickets: 1 `updated:` before tag date, 2 after → only 2 collected; 1 ticket with absent `updated:` → fallback `find -newer` collects it if mtime after tag |

| FR-5 | Integration | Slug `feat-add-login` → `feat`; slug `my-feature` → `other`; commit `chore(ci): update` → `chore` |
| FR-6 | Integration | Title "Add login page" + commit "feat: add login page" → single `feat` entry; title "Fix: off-by-one" vs commit "fix: offbyone" → NOT deduplicated |
| Sanitizer | Integration | Commit subject `[Unreleased] - 2026-06-21` → sanitized to `\[Unreleased\] - 2026-06-21` in output, does not trigger idempotency replacement; commit subject with `\n## heading` → newline stripped, single-line output |
| FR-7 | Integration | Output has `## [Unreleased]`, subsections present only when non-empty |
| FR-8 | Integration | CHANGELOG absent → created; existing with versioned releases but no `[Unreleased]` → block prepended; existing with `[Unreleased]` as last block (no subsequent `## ` heading) → replaced through EOF |
| FR-9 | Integration | Run twice → single `## [Unreleased]` block; runtime warning printed to stderr on second run |
| FR-9 (edge) | Integration | HEAD == tag (zero commits, zero new tickets) → empty or minimal unreleased block, no crash |
| FR-10 | Integration | stdout matches `CHANGELOG.md` prepended block |
| Portability | Integration | Command runs without error on macOS (BSD `touch`) and Linux (GNU `touch`) |
| Shell safety | Integration | Tag name containing spaces does not cause word-splitting or command injection |

## Tradeoffs

- **Chose slug-prefix heuristic**: avoids schema change; many tickets land in `other`; follow-on can add `category:` field.
- **Chose command-file / bash over Python**: lower cost; compensated by shell safety requirements and integration tests.
- **Date-only comparison**: tickets updated same day as tag may be included or excluded; acceptable for v1.
- **Clobber risk on manual `[Unreleased]` edits**: operator warned at runtime via stderr; not fixing in v1.
- **Markdown escaping (not HTML-encoding)**: backslash-escaping `[`, `]`, `(`, `)` is idiomatic in Markdown lists and renders correctly in all standard parsers; HTML-encoding (`&#91;`) would look wrong in source view.

## Risks

- `updated:` hand-edited to wrong date → ticket excluded/included incorrectly; field is authoritative.
- No conventional commits and no typed slugs → `other`-only changelog; acceptable.
- `touch -t` format construction (`%at` epoch → `[[CC]YY]MMDDhhmm[.SS]`) must be verified on macOS — add to integration test portability scenario.
